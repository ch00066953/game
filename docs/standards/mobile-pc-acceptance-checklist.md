# 移动端 + PC 统一验收清单

用途：所有游戏改动都按本清单做验收，避免出现“PC 可玩、移动端不可玩”或反向问题。

## 一、设备与视口

必测视口：
- 390 x 844（手机竖屏）
- 768 x 1024（平板）
- 1280 x 720（PC）

验收项：
1. 主界面无横向滚动条
2. 游戏区域完整可见
3. HUD 不遮挡核心操作区域
4. 弹窗可完整显示并可关闭

## 二、输入一致性

验收项：
1. PC 键盘路径：开始、游玩、暂停、结算、重开可闭环
2. 移动触控路径：开始、游玩、暂停、结算、重开可闭环
3. 方向输入不冲突（防反向、防连触异常）
4. 长按与连点行为符合预期
5. 误触不会触发页面滚动或浏览器手势冲突

## 三、可读性与可点击性

验收项：
1. 核心文本在手机端可读
2. 关键按钮在手机端易点（避免过小）
3. 颜色对比足够，状态可区分
4. 失败/胜利提示明确且可恢复

## 四、性能与稳定性

验收项：
1. 连续游玩 3-5 分钟无明显卡顿
2. 无持续增长的内存或对象泄漏迹象
3. 页面切后台再回来，游戏状态可控（继续或重置符合设计）
4. 控制台无新增报错

## 五、玩法回归

验收项：
1. 核心机制无回退（移动、碰撞、判定、计分）
2. 关卡或敌人行为符合设计
3. 暂停与重开不会破坏状态
4. 存档功能（如有）可读写且兼容旧数据

## 六、发版前快速检查（10 分钟）

1. 手机视口走一遍完整流程
2. PC 视口走一遍完整流程
3. 检查 README 控制说明是否与当前版本一致
4. 记录本次已知问题与规避方案

## 使用方式

- 每次提交前，按本清单做最小验收。
- 发现不通过项时，先修复 P0（无法游玩、阻断流程）再处理 P1（体验类问题）。
- 新游戏立项时，把本清单链接放进该游戏 README。
