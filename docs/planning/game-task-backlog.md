# 游戏开发任务池（按优先级）

说明：本任务池聚焦规划与执行拆分，默认每项都需要移动端与 PC 端同步验收。

## P0：立即推进

1. 打砖块 MVP
- 目标：7 天内可玩版本
- 范围：挡板、反弹、砖块血量、通关/失败、重开
- 验收：键盘与触控都可操作挡板，移动端无遮挡

2. 2048 MVP
- 目标：3-5 天可玩版本
- 范围：4 向滑动、合并规则、分数、失败判定、重开
- 验收：触控滑动方向稳定，PC 键盘方向键稳定

3. 推箱子 MVP
- 目标：5-7 天可玩版本
- 范围：地图加载、推动规则、关卡完成判定、重置
- 验收：移动端按钮布局不遮挡地图

4. 通用输入层规范文档
- 目标：沉淀键盘与触控映射标准
- 范围：方向、确认、取消、暂停、灵敏度阈值
- 验收：新游戏可直接复用

## P1：两周内启动

1. 泡泡龙简化版
- 目标：核心发射与碰撞可玩
- 风险：碰撞与消除性能
- 建议：先做固定关卡，再做随机布局

2. 塔防基础版
- 目标：1 张地图 + 2 塔 + 3 敌人
- 风险：寻路与帧率稳定
- 建议：先固定路径，后扩展自由建造

3. 平台跳跃进阶版
- 目标：移动、跳跃、碰撞、死亡重生、终点
- 风险：移动端虚拟按键手感
- 建议：按钮大小与触控区先行定标

## P2：中期储备

1. 卡牌 Roguelike 原型
- 范围：回合制、抽牌、出牌、敌人行动、战后奖励
2. 战棋小地图
- 范围：网格寻路、行动点、攻击范围、回合切换
3. 轻量 RPG
- 范围：角色成长、背包、任务、存档

## 统一 Definition of Done

1. 功能完成：主玩法循环可闭环
2. 双端完成：触控与键盘路径都可达成通关
3. 体验完成：关键反馈到位（音效/动画/提示）
4. 文档完成：README 与变更记录已更新
5. 回归完成：通过双端验收清单
